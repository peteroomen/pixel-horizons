## Goal

A deployed, CI-green Next.js app whose single page renders a pixel-perfect PixiJS test scene — crisp at any window size on desktop and phone — with all tooling (lint, type-check, test, pre-commit hooks) working.

## Approach

Scaffold into the EXISTING repo at `~/personal/pixel-horizons` (it already contains `CLAUDE.md` and `docs/` on main — do not `create-next-app` into a subdirectory; scaffold in place or in a temp dir and merge files in).

- **Node 22 required** (`nvm use 22`) — Husky/lint-staged break on older Node. Same stack as `~/personal/breakout-roguelite` and `~/personal/perihelion`; check their CLAUDE.md "Known issues" sections before fighting any tooling error (8bitcn/ui + shadcn CLI quirks, `next lint` removed in Next.js 16 → use `eslint src`, Vercel CLI env vars).
- Next.js 16 App Router, TypeScript strict, Tailwind v4, 8bitcn/ui, PixiJS v8, Motion, Vitest, Husky + lint-staged, Prettier.
- **Virtual resolution 640×360** (16:9), integer-scaled to fit the window with letterboxing, nearest-neighbor scaling everywhere (PixiJS v8: set default texture scale mode + `roundPixels`). This is the project's foundational rendering contract — get it right here.
- `GameCanvas.tsx` owns the PixiJS `Application` lifecycle: async `init()`, proper `destroy()` on unmount, **safe under React 19 StrictMode double-mount** (no duplicate canvases, no leaked WebGL contexts). Test scene: checkerboard background + a few sprites, one animated/moving at constant speed (visual tearing/blur check), current scale factor displayed.
- Directory skeleton per CLAUDE.md module architecture (`src/game/sim/`, `src/game/surface/`, `src/game/data/`, `src/renderer/`, `src/components/`) with placeholder files where needed — but NO game logic.
- Vitest configured with one trivial passing smoke test (real sim tests are Slice 1.2).
- Scripts: `dev`, `build`, `lint` (eslint src), `type-check`, `test`. Pre-commit: lint-staged (eslint + prettier) — resolve node_modules from git-common-dir so hooks work in worktrees (see breakout's hook for the pattern).
- Deploy to Vercel, hooked to the GitHub repo for preview deploys on PRs.

## What actually happened

Went largely to plan. Production URL: **https://pixel-horizons.vercel.app**

- Scaffolded with `create-next-app` (Next 16.2.9 / React 19.2.4) in a temp dir, merged in, preserving CLAUDE.md and docs/.
- 8bitcn/ui: as documented in Perihelion's known issues, the shadcn CLI was skipped — base button written manually (plain `<button>`, NOT Perihelion's `@base-ui/react` dependency, to avoid adding a library), 8bit button + retro.css ported from Perihelion.
- Scaler decision: integer zoom is computed in **device pixels** (CSS size × devicePixelRatio), so ×2 on a retina laptop means 2 physical px per virtual px — genuinely crisp. Canvas backing store = 640×zoom, CSS size = backing/dpr.
- Sub-1× edge case: **fractional downscale chosen** (over min-1×-with-scroll) so the scene always fits on phones; documented in `pixel-scale.ts`.
- StrictMode safety via `cancelled` flag racing async `Application.init()` against effect cleanup; verified in browser: exactly 1 canvas, zero console warnings.
- Verified in headless browser: ×2 at desktop (1280×720 backing @ dpr 2), ×1 at mobile preset (375px), ×0.88 at 280px-wide viewport; pre-commit hook blocks a staged lint error (tested with a deliberate `any` + unused var).
- `pnpm lint`, `type-check`, `test` (3 tests), `build` all green.
- Prettier reformat churn on hand-written docs/ was reverted; `docs/` added to `.prettierignore`.
- Vercel project created in the same team as siblings, GitHub repo connected for PR preview deploys, deployed via CLI. Note: only the canonical URL is public; `*-peter-oomens-projects.vercel.app` deployment URLs are behind Vercel Authentication (401) — expected.
