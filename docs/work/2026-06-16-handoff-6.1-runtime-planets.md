## The three PixiJS v8 gotchas you WILL hit (the spike's real payoff)

These cost most of the spike's debugging. The fixes are already in `planet-spike.ts` ‚Äî copy them:

1. **GLSL ES 3.00 needs the literal header.** Custom shader source must start with
   `#version 300 es` or `in`/`out`/interface blocks won't compile ‚Äî `GlProgram.from` greps for
   that exact string (`isES300`).
2. **`Shader.from` resource uniforms are LOOSE in the GL path.** Declare `uniform float uSeed;`,
   NOT a UBO `uniform Block { ... };`. A block leaves an active UBO with no buffer bound ‚Üí
   `gl INVALID_OPERATION` and a **silently skipped draw** (no console error ‚Äî only `gl.getError()`
   + reading pixels reveals it).
3. **Pixi feeds camera UBOs only to its own mesh shader.** A custom mesh shader doesn't reliably
   get `uProjectionMatrix`/`uTransformMatrix`. The spike sidesteps this with **clip-space (NDC)
   quad corners**. For 6.1 the plan recommends **baking each planet to a `RenderTexture`** and
   composing it as a normal `Sprite` ‚Äî that dodges the issue entirely and plays nice with the
   integer-zoom pipeline.

Bonus gotcha: in this worktree, **Turbopack served stale shader bundles** through HMR/reload ‚Äî
a `preview_stop`/`start` (dev-server restart) was needed to pick up shader edits. Budget for it.

## Determinism contract (don't break it)

The shader is a pure function of `uSeed` (+ cosmetic `uTime`). Feed `uSeed` from the seeded RNG
(`rng.ts`) via a deterministic planet descriptor on the `'planet'` map node, carried on
`RunState`. Verified byte-reproducible in the spike. Same run seed must yield the same planet in
map / orbit / surface ‚Äî keep identity in the sim, pixels in the renderer.
