# Slice 3.4 — Clone death + surface enemies + Rocky hazards

**Date:** 2026-06-15
**Branch:** claude/tender-allen-kgp133
**Roadmap item:** Phase 3 — 3.4 Clone death + hazards

## Goal

Surface drops gain real stakes: the clone has HP, can be hurt by enemies and
hazards, and dies — dropping its backpack as a corpse to be recovered, with a
free-first / Scrap re-print loop. The full GDD §6.7–6.10 threat roster lands in
one PR (per the scope decision): three surface enemies, three Rocky hazards, the
Shield Bubble turning on, and the death/corpse/re-print sequence.

## Approach

All new mechanics live in the React/Pixi-free surface sim (`src/game/surface/`),
unit-tested under Vitest. The renderer and React layer are pure presentation fed
by the existing `SurfaceView` snapshot + the renderer reading `SurfaceState`.

Key decisions made upfront:

- **Threats come from explicit level tile-tokens**, not procedural spawn weighting
  (that's a map-gen concern, deferred). Matches the GDD §6.9 token grammar and keeps
  the surface sim deterministic with no RNG threaded in. New tokens: `H` Bloom Hopper,
  `G` Scrap Grubber, `C` Ceiling Dropper, `^` Spike Bramble, `~` Crumbling Sandstone,
  `V` Sandstorm Vent.
- **Clone HP / re-prints are ephemeral per-drop state**, not persisted to RunState
  (saves happen at node boundaries, not mid-drop) — so **no `RUN_STATE_VERSION` bump**.
  The only run-state mutation is spending banked Scrap on a paid re-print.
- **Re-print economy stays in the orchestrator** (3.2 invariant — `clone.ts`/surface
  sim are economy-free). The sim exposes `clone.dead` + `reprintsUsed` + a pure
  `reprintClone(state)` that respawns; `main.ts` gates it against `run.resources.scrap`
  (first re-print per visit free, then `REPRINT_SCRAP_COST`) and enriches the emitted
  view with `reprintScrapCost`/`canReprint`.
- **Matrix HP/melee become structured data**: new `PlanetItemEffect` kinds `clone-hp`,
  `melee-damage`, `clone-regen` projected into `CloneCapabilities` (`maxHp`,
  `meleeDamage`, `regenMsPerHp`). Standard=3HP, Scavenger/Enforcer/Assault=2HP,
  Enforcer +1 melee, Repair slow regen.
- **Shield Bubble**: absorbs one hit (incl. hazards), pops, recharges over its
  `cooldownMs` (pragmatic read of "no passive recharge" — documented deviation).
- **All numbers in `data/surface.ts` / `data/economy.ts`** — zero magic numbers in logic.

## What actually happened

Built the full slice in one PR as scoped. Sim core first (fully Vitest-covered),
then renderer, React overlay, and the seeded test level.

- **Clone combat** (`clone.ts`): HP/i-frames/hit-stun/knockback, shield absorb,
  regen, `damageClone`/`respawnClone`, and an `externalVx` param so vents push the
  clone through the AABB collision pipeline (not a post-hoc nudge).
- **Enemies** (`enemies.ts`): Hopper (patrol + leap), Grubber (passive→aggro,
  cornered detection on a post-move wall hit), Dropper (ceiling cling → drop →
  landing stun → shield-pop fling). One-hit-per-swing via an enemy `hitCooldown`.
- **Hazards** (`hazards.ts`): bramble overlap, Crumbling Sandstone state machine
  (toggles `map.tiles` + `version`), Sandstorm Vent cycle + push band.
- **Drops/corpse** (`drops.ts` + `surface.ts`): enemy drops → backpack, overflow
  → magnetised world items; death drops a single-instance corpse; corpse-run
  recovery; stranded-while-dead surfaces the corpse loot as LOST.
- **Economy**: `reprintClone` in the sim is pure; `main.ts` gates free-first /
  `REPRINT_SCRAP_COST` against `run.resources` and enriches `canReprint` (ADR 009).
- **Renderer**: enemies, world items, corpse + off-screen beacon, bramble/
  crumbling/vent tiles, shield ring, i-frame blink, hit red-flash + camera shake,
  death fade. Screen overlays live in a stage-level container (not camera-offset).
- **UI**: HUD HP pips + shield status; Cloning Bay overlay (Re-print free/N-scrap
  + Abandon).

Decisions captured in **ADR 009**. No `RUN_STATE_VERSION` bump (clone/re-print
state is ephemeral per-drop). Shield recharges on cooldown (documented deviation
from the GDD's "no passive recharge" line).
