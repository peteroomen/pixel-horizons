# 6.1 slice 3 ŌĆö Atmosphere/sky recolor (Option A)

**Date:** 2026-06-19
**Branch:** feature/atmosphere-sky-recolor
**Roadmap item:** Phase 6 ŌĆö 6.1 Visual identity pass (runtime planets, slice 3)

## Goal

The sky gradient, horizon mesas in `skyLayer` recolor to match the planet's hue family, so
a green planet no longer drops the player under a fixed warm-orange sky.

## Approach

`skyLayer(g, W, H)` in `src/renderer/surface-sprites.ts` is the only hardcoded-warm layer
remaining after slice 2. `surfaceBackdrop(W, H, P)` already passes the terrain ramp `P` to
`midLayer`/`nearLayer`; this slice threads it into `skyLayer` too.

Key decisions:
1. **Sky ramp ŌēĀ land ramp.** The land ramp is for rock; a sky wants a light, desaturated
   gradient (horizon glow ŌåÆ zenith). A new `skyRampFor(descriptor): Ramp` in `palette.ts`
   is added next to `surfaceRampFor`. It returns a light R64 ramp derived from the planet's
   hue family. Do NOT reuse the land ramp directly ŌĆö it reads muddy.
2. **Sun stays fixed warm.** Tinting the sun per star colour is scope creep (star/space-bg
   generator is a later slice). The sun disc stays its warm yellow.
3. **Mesas take the land ramp's lighter steps** (atmospheric perspective: distant = lighter,
   less saturated). This ties the sky and ground together visually.
4. **Sky must stay clearly lighter than terrain** so platform silhouettes remain legible.
   Tested mentally against the dark tundra ramp (tightest contrast case).

Sky ramp design (6-step, lightest ŌåÆ darkest, R64-locked):
- Per hue family, pick 6 R64 steps that read as a light atmospheric sky gradient.
- Index 0 = brightest zenith; index 5 = deepest near-horizon glow.
- Keep all steps lighter/more desaturated than the corresponding land ramp steps.

Sky ramps by land ramp family:
- verdant green ŌåÆ soft teal/cyan sky (`#8ff8e2..#0eaf9b` family)
- tundra grey-green ŌåÆ cool grey-blue sky (`#c7dcd0..#484a77` family)
- rust desert ŌåÆ warm peach sky (the existing orange ŌĆö `#fdcbb0..#cd683d`, keeps current look)

## What actually happened

Built exactly as planned. Key decisions:

- **Sky ramp design:** Three sky ramps (teal/cyan for verdant, cool grey-blue for tundra, warm
  peach for rust/desert) keyed by land ramp index. The lookup uses the same `hash01(seed + 1)`
  call as `planetRamps` to stay in sync with the land-ramp selection ŌĆö no extra hash needed.
- **Mesa colors:** Horizon mesas now use `mix(P[1], P[2], 0.4)` from the *land* ramp's lighter
  steps (atmospheric perspective: distant hills read lighter/less saturated). Mesa lit tops use
  `S[1]` from the sky ramp, tying sky and ground together.
- **Sun stays fixed warm** ŌĆö the `mix('#fbff86', '#fbb954', 0.3)` blob is unchanged. Star/sun
  tinting is explicitly out-of-scope for this slice.
- **`ROCKY_SKY` constant** exported from `surface-sprites.ts` as the default sky ramp so
  `skyLayer`/`surfaceBackdrop` still work when called without a sky ramp (no callers break).
- **Full chain threaded:** `skyRampFor` in `palette.ts` ŌåÆ `main.ts` `enterSurface` ŌåÆ 
  `SurfaceModeOptions.skyRamp` ŌåÆ `createSurfaceRenderer(app, terrainRamp, skyRamp)` ŌåÆ 
  `surfaceBackdrop(W, H, terrainRamp, skyRamp)` ŌåÆ `skyLayer(g, W, H, S, P)`.
- All 4 `skyRampFor` tests pass (determinism, R64-lock, distinct families, sky brighter than land).
- Test count: 516 ŌåÆ 520 (4 new tests).
