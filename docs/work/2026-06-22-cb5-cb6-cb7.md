# CB.5 + CB.6 + CB.7 — Ball Glyph Grammar, Bloom Interference, Platformer Cleanup

**Date:** 2026-06-22
**Branch:** `claude/handoff-review-cb-tasks-vv5var`
**Roadmap items:** Phase CB — CB.5, CB.6, CB.7

## Goal

One PR delivers all three remaining Core Breaker slices: (1) ball glyph grammar + dual-face module surface info in the HUD and Workbench, (2) Bloom interference soft-fail that drains banked yield if growths are ignored, (3) full platformer code removal.

## Approach

### CB.5 — Ball glyph grammar + dual-face module surface info

The GDD §6.4 says each ball reads at a glance via a trajectory glyph (straight/arc/curve) plus a count badge, with no text required. A module should show both its combat face (cards) and its surface face (ball glyph + type) on one card at 375px.

Ball → glyph mapping (§6.4 UI law):
- `drill` → **straight** (bores through — pierce trajectory)
- `standard`, `heavy`, `split` → **arc** (bouncy)
- `ghost` → **curve** (phases through, gently curving)

Implementation plan:
- Add `BALL_GLYPH: Record<BallType, 'straight' | 'arc' | 'curve'>` to `src/game/data/core-breaker.ts`
- Add `ballFace: { type: BallType; glyph: 'straight' | 'arc' | 'curve' } | null` to `ShipModuleView` in `ship-view.ts`; populate it from `projectMiningRoster` in `buildShipView`
- Create `src/components/foundry/BallGlyphIcon.tsx` — CSS-drawn SVG glyphs for straight/arc/curve + label
- Update Workbench `ModuleRow`: add a compact "surface face" row (glyph + ball type label) below the toggle area
- Update `CoreBreakerHud`: add a small count badge overlay (`×N`) on each ball thumbnail in the queue preview

### CB.6 — Bloom interference (soft-fail)

GDD §6.5 / §6.10: "a growth that must be cleared with a shot, or it starts eating banked yield."

The drain lives at the **renderer layer** (not the sim) because it persists across shots and is not determinism-critical. The sim stays untouched.

- Add tunable constants `BLOOM_DRAIN_THRESHOLD = 3` and `BLOOM_DRAIN_PER_SHOT = 1` to `src/game/data/mining-run.ts`
- In `core-breaker-renderer.ts`: track `bloomShotsActive` (reset when zero bloom pegs remain). After the threshold, subtract `BLOOM_DRAIN_PER_SHOT` Scrap from `banked.scrap` at the end of each shot. Clamp at 0.
- Add `bloomThreat: number` (integer shots-over-threshold, 0 = safe) to `CoreBreakerHudState`; emit `onHud` when it changes
- HUD header: a compact amber chip "BLOOM ▲N" that appears when `bloomThreat > 0`, fades when 0
- Tests: at least 3 covering threshold behaviour, yield drain, and clear-resets
