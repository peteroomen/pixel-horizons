# 6.1 slice 2 — Surface-tileset recolor (biomes match the planet)

**Date:** 2026-06-18
**Branch:** feature/surface-recolor
**Roadmap item:** Phase 6 — 6.1 Visual identity pass (runtime planets, slice 2). Coordinates with 5.3.

## Goal

The terrain you drop onto is tinted to match the planet you just saw from orbit — same
seed+node ⇒ same planet ⇒ same surface tint. The ADR 002 §3 / ADR 010 "biomes match the
planet" payoff, wired through the existing planet ramp.

## Approach

Both halves already exist and were built to meet here (see handoff
`2026-06-18-handoff-6.1-slice2-surface-recolor.md`). This slice is the seam:

1. **One seam helper** in `renderer/palette.ts`: `surfaceRampFor(descriptor): Ramp`. For the
   only type today (`terran`) it returns `planetRamps(descriptor.seed).land`. 5.3 branches here
   for Volcanic/Ice. Keeps `palette.ts` the single source of the ramp.
2. **Carry the descriptor to the surface.** `enterOrbit` already computes
   `planetForNode(run.seed, nodeId)`. Store it as `currentPlanet` in `main.ts`; `enterSurface`
   uses it so the ground matches the orbit the player just saw (critical for the `?mode=orbit`
   dev path, whose `'dev-orbit'` node id is *not* `run.position.nodeId` — recomputing from the
   node id would desync). Fall back to `planetForNode(run.seed, 'dev-surface')` for the
   no-orbit `?mode=surface` path.
3. **Thread the ramp, not the descriptor, into the renderer.** Add `terrainRamp: Ramp` to
   `SurfaceModeOptions`; `surface-mode.ts` passes it to `createSurfaceRenderer`. Keeps the
   renderer ramp-driven and `surface-sprites.ts` a pure factory.
4. **Recolor the terrain tiles** in `surface-renderer.ts` by passing the ramp into the tile
   factories. `rockFillTile`/`rockSurfaceTile` already take a ramp; add the same `P: Ramp`
   param to `crumbleTile`, `brambleTile` (rock floor only), and the rock-debris FX
   (`fxShatterSprite`, `fxDustSprite`) so debris matches the dirt.

**Scope decision — terran only.** `planets.ts` has one type (`terran`); Volcanic/Ice planet
*types* + their hazards are 5.3. This slice ships the recolor pipeline + terran; 5.3 adds types
by extending `surfaceRampFor`'s switch.

**What stays fixed (non-terrain signals/props):** bedrock (cool = unbreakable grammar),
biomineral vein, scrap cache, scanned vein, core crystal (the lone pure-cyan glint). These are
material/signal grammar, not biome tint — recoloring them would break the "two palettes, one
law" read. The bramble *thorns* (Bloom) and crystal sockets stay too; only their rock host
recolors.

## What actually happened

Built the seam exactly as planned, then **extended scope to the cave-mouth backdrop rock**
after browser verification showed the defined tiles-only recolor was barely visible — the test
level (`ROCKY_TEST_LEVEL`) is ~90% `#` bedrock (fixed slate by the "cool = can't mine" grammar)
and the dominant on-screen surface is the `surfaceBackdrop` cave-mouth, which was fixed rust. So
a green planet still dropped into a rust cave. With the user's go-ahead, threaded the ramp into
`surfaceBackdrop`'s **rock layers** (`midLayer` columns + `nearLayer` cave frame, the latter
mixed toward black for a biome-tinted dark vignette); **sky + sun + horizon mesas stay fixed**
(atmosphere — its own later slice).

Decisions made along the way:

- **Deposit tiles' rock host recolors too.** `biomineral/scrap/scanned/core/hidden` embed a
  `rockFillTile()` body; leaving it rust while the field recolored made deposits read as rust
  patches in a green biome. Rule landed on: **rock host recolors, embedded signal stays fixed**
  (crystal socket, scrap box, cyan vein/glint, bramble thorns). **Bedrock stays fully fixed** —
  it is a separate cool material, not a biome host.
- **Rock-debris FX** (`fxShatterSprite`, `fxDustSprite`) recolor with the ramp; the pure-orange
  mining-impact spark stays fixed (the one action accent).
- **Verification was a detour:** the orbit/surface looked unchanged at first; pixel-reading the
  generated tile canvas proved the recolor was correct per seed (tundra `#b2ba90` vs verdant
  `#cddf6c`) and the renderer received the right ramp — the muted *visible* result was purely
  the bedrock-heavy level + fixed backdrop, which the backdrop work then addressed.

Browser-verified (`?mode=orbit`): verdant7 → green/teal cave, abc123 → grey-green tundra cave,
both matching their orbit planet and reproducible by `?seed=`; `?mode=surface` dev fallback
(`dev-surface` descriptor) renders without crash; 375px legible. No `RunState`/version change.
