# Runtime Planets — 6.1 slice 1 (generation + run integration)

**Date:** 2026-06-16
**Branch:** feature/runtime-planets
**Roadmap item:** Phase 6 — 6.1 Visual identity pass (first slice; runtime planet generation)
**Builds on:** ADR 010 (runtime planet generation), the spike
(`docs/work/2026-06-16-planet-shader-spike.md`)

## Goal

A `'planet'` node in a run shows a **real, runtime-generated pixel planet** that is
deterministic from the run seed and palette-locked to Resurrect 64 — productionising the spike
and wiring planet identity through the sim. One planet type (terran) is enough for this slice;
the system is built to take more.

## Approach

Promote the throwaway spike into a real renderer module + a sim-side planet descriptor, then
show the planet at one real surface in-game. Keep the blast radius small: no surface-tileset
recolor, no extra planet types, no backdrops yet (those are later 6.1 slices, the recolor
coordinated with 5.3).

Key decisions (carried from ADR 010 / the spike):

- **Determinism via uniform, not storage.** Planet identity = a small descriptor
  (`{ seed, type, ramp }`) derived from the run seed through `rng.ts`; the renderer feeds
  `seed` as a `uSeed` uniform. Statics baked to a `RenderTexture` once per planet.
- **Palette as uniforms.** `palette.ts` owns the Resurrect 64 set + per-planet ramp
  generation/validation; ramps are passed to the shader as uniforms (ADR 010), not a post LUT.
- **Loose uniforms + `#version 300 es` + clip-space (or RT bake)** per the spike's three v8
  gotchas. Decide scene-graph integration vs render-to-texture here; default to **RT bake**
  (a `Sprite` of a baked planet texture composes cleanly with the existing Pixi scene and the
  integer-zoom pipeline, and sidesteps the custom-shader camera-UBO issue).
- **Sim stays renderer-free.** The planet descriptor lives in `sim/` (unit-testable); the
  shader/ramps live in `renderer/`. `palette.ts` is pure data+math (testable), no Pixi import
  for the ramp logic.

## What actually happened

Slice shipped end to end and browser-verified. A planet node now arrives into a new **orbit
screen** showing the runtime-generated planet (`TERRAN` + DROP), and DROP proceeds into the
normal surface flow.

- **palette.ts** — canonical Resurrect 64 (fetched exact 64 hex from Lospec) + `planetRamps`
  (6-step swap-compatible ocean/land ramps) + `snapToPalette`. Pure, 9 tests.
- **data/planets.ts + sim/planet.ts** — `planetForNode(seed, nodeId)` descriptor, **derived
  not stored** (no `RUN_STATE_VERSION` bump), 4 tests.
- **renderer/planet.ts** — `renderPlanetTexture` bakes to a `RenderTexture`; R64 ramps as loose
  `vec3[6]` uniforms; hard `discard` halo fix.
- **Orbit screen** — `orbit-mode` + `orbit-renderer` (planet + deterministic starfield), new
  `'orbit'` GamePhase, `OrbitView` + `onOrbitUpdate` callback, `dropToSurface()` handle, DOM
  overlay (planet name + DROP). Planet arrival routes `lane → orbit → (DROP) → surface`.
- **`?mode=orbit`** dev knob (mirrors `?mode=surface`) for verifying orbit without a lane.

Decisions/changes vs. plan: descriptor **derived** rather than stored on RunState (cleaner,
no version bump); surface-tileset recolor stays slice 2 (with 5.3) as agreed.

Two real bugs found + fixed: (1) `Shader.from` resource uniforms are **loose** in Pixi's GL
path — a UBO block declaration → silent `INVALID_OPERATION`, skipped draw (spike). (2) large
sim seeds (~1e6) overflow float32 in the shader hash and flatten terrain → variety moved to a
bounded CPU-computed `vec2` noise offset.

512 tests green; lint + type-check clean. `/planet-spike` dev harness shows 7 distinct
deterministic planets; `?mode=orbit` shows the orbit screen; DROP reaches the surface.
