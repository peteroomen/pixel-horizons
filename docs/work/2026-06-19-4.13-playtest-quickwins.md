# Playtest Quick-Wins Bundle

**Date:** 2026-06-19
**Branch:** feature/playtest-quickwins
**Roadmap item:** Phase 4 — 4.13 Playtest quick-wins bundle

## Goal

Land four cheap, independent playtest pain-point fixes in one PR: buy duplicate
modules with 1-stock-per-shop, remove the Anchormaw pay-toll, add a lane progress
bar to the combat screen, and tighten card legibility (louder title, card type,
consistent width). Bigger sibling work (depth difficulty, full card rework) stays
in 4.10/4.11/6.8.

## Approach

Four self-contained changes, each with its own commit:

1. **Buy duplicates / 1 stock per shop** (from 4.10). Today `station-view.ts`
   greys out owned modules (`owned` flag → no buy) and `shop-inventory.ts` has no
   purchased-stock tracking. Drop the owned-gate so you can re-buy a module for
   more copies of its cards; add per-shop "sold out" state so each component is
   stock-1 (buying it removes that offer). Stock is **derived deterministically**
   from (seed, sector, nodeId) + which of this shop's offers the run has already
   purchased — tracked on `RunState` (a small `purchasedOffers` set keyed by
   `sector:nodeId:moduleId`), persisted across resume per ADR 005. No RNG re-roll.

2. **Remove the pay-toll mechanic** (from 4.10). Anchors must be killed. Resolve
   the GDD §5.7/§5.1 design note first (the toll is documented as "the counter to
   engine builds"): the anchor stays a hard must-kill wall; the engine-build
   counter is simply that anchored lanes can't be skipped by arrival, so engine
   builds still can't run past them. Then rip out `payToll`/`canPayToll` and the
   `TOLL PAID` state across the stack. The `anchor` field stays on `CombatView`
   (the latch is still shown — "LANE BLOCKED — destroy to pass") but loses
   `tollScrap`/`payable`.

3. **Lane progress bar** (from 4.11). The data already exists —
   `CombatView.travel = { progress, distance }` is populated whenever
   `state.lane !== null`. Pure UI: a thin segmented bar at the top of the combat
   screen showing distance travelled vs lane length, hidden outside a lane. No sim
   change.

4. **Card legibility** (from 6.8). In `CombatCard.tsx`: louder title (bigger,
   higher-contrast header type), a visible **card type** chip (ATTACK / SKILL /
   POWER), and a consistent card width so the hand reads as a clean row. Derive the
   type in `combat-view.ts` from card shape (damage-dominant → ATTACK;
   applies-a-status/Exhaust buff → POWER; shield/travel/utility/draw → SKILL) and
   expose `CardView.cardType`. Malfunction/infested/discard states keep their
   existing tag.

## What actually happened

All four changes landed as planned. One design deviation in change 4: the initial
`deriveCardType` included `exhaust === true` as a POWER signal, but Emergency Barrier
(exhaust, temp-shield, no status) should be SKILL — rule tightened to `applies-status`
only for POWER. Four commits, one per change.

**Save version decision:** bumped `RUN_STATE_VERSION` 3→4 for `purchasedOffers`. The
deserializer rejects v3 saves (clean break — old saves degrade to "no save").

**Anchor type simplified:** `anchor?: { tollScrap: number }` → `anchor?: true` in
`EnemyDef` and `anchor: { blocked: true } | null` in `CombatView`.
